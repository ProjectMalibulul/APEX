					<div id="mw-content-text" class="mw-body-content"><div class="mw-subjectpageheader">
</div><div class="mw-content-ltr mw-parser-output" lang="en" dir="ltr"><div class="shortdescription nomobile noexcerpt noprint searchaux" style="display:none">Graph drawing with vertices in horizontal layers</div><figure class="mw-default-size" typeof="mw:File/Thumb"><a href="/wiki/File:DC%2B%2B_derivatives.svg" class="mw-file-description"><img src="//upload.wikimedia.org/wikipedia/commons/thumb/7/7e/DC%2B%2B_derivatives.svg/500px-DC%2B%2B_derivatives.svg.png" decoding="async" width="340" height="124" style="--mw-file-upright: 1.35" class="mw-file-element mw-file-upright" srcset="//upload.wikimedia.org/wikipedia/commons/thumb/7/7e/DC%2B%2B_derivatives.svg/960px-DC%2B%2B_derivatives.svg.png 2x" data-file-width="1803" data-file-height="655" /></a><figcaption>A layered drawing of a <a href="/wiki/Directed_acyclic_graph" title="Directed acyclic graph">directed acyclic graph</a> produced by <a href="/wiki/Graphviz" title="Graphviz">Graphviz</a></figcaption></figure>
<p><b>Layered graph drawing</b> or <b>hierarchical graph drawing</b> is a type of <a href="/wiki/Graph_drawing" title="Graph drawing">graph drawing</a> in which the <a href="/wiki/Vertex_(graph_theory)" title="Vertex (graph theory)">vertices</a> of a <a href="/wiki/Directed_graph" title="Directed graph">directed graph</a> are drawn in horizontal rows or layers with the <a href="/wiki/Glossary_of_graph_theory#edge" title="Glossary of graph theory">edges</a> generally directed downwards.<sup id="cite&#95;ref-dett98&#95;1-0" class="reference"><a href="#cite_note-dett98-1"><span class="cite-bracket">&#91;</span>1<span class="cite-bracket">&#93;</span></a></sup><sup id="cite&#95;ref-bm01&#95;2-0" class="reference"><a href="#cite_note-bm01-2"><span class="cite-bracket">&#91;</span>2<span class="cite-bracket">&#93;</span></a></sup><sup id="cite&#95;ref-hn14&#95;3-0" class="reference"><a href="#cite_note-hn14-3"><span class="cite-bracket">&#91;</span>3<span class="cite-bracket">&#93;</span></a></sup> It is also known as <b>Sugiyama-style graph drawing</b> after <a href="/wiki/Kozo_Sugiyama" title="Kozo Sugiyama">Kozo Sugiyama</a>, who first developed this drawing style.<sup id="cite&#95;ref-4" class="reference"><a href="#cite_note-4"><span class="cite-bracket">&#91;</span>4<span class="cite-bracket">&#93;</span></a></sup>
</p><p>The ideal form for a layered drawing would be an <a href="/wiki/Upward_planar_drawing" title="Upward planar drawing">upward planar drawing</a>, in which all edges are oriented in a consistent direction and no pairs of edges cross. However, graphs often contain cycles, minimizing the number of inconsistently oriented edges is <a href="/wiki/NP-hard" class="mw-redirect" title="NP-hard">NP-hard</a>, and minimizing the number of crossings is also NP-hard; so, layered graph drawing systems typically apply a sequence of <a href="/wiki/Heuristic" title="Heuristic">heuristics</a> that reduce these types of flaws in the drawing without guaranteeing to find a drawing with the minimum number of flaws.
</p>
<meta property="mw:PageProp/toc" />
<div class="mw-heading mw-heading2"><h2 id="Layout_algorithm">Layout algorithm</h2><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Layered_graph_drawing&amp;action=edit&amp;section=1" title="Edit section: Layout algorithm"><span>edit</span></a><span class="mw-editsection-bracket">]</span></span></div>
<p>The construction of a layered graph drawing proceeds in a sequence of steps:
</p>
<ul><li>If the input graph is not already a <a href="/wiki/Directed_acyclic_graph" title="Directed acyclic graph">directed acyclic graph</a>, a set of edges is identified the reversal of which will make it acyclic. Finding the smallest possible set of edges is the <a href="/wiki/NP-complete" class="mw-redirect" title="NP-complete">NP-complete</a> <a href="/wiki/Feedback_arc_set" title="Feedback arc set">feedback arc set</a> problem, so often <a href="/wiki/Greedy_algorithm" title="Greedy algorithm">greedy heuristics</a> are used here in place of exact optimization algorithms.<sup id="cite&#95;ref-dett98&#95;1-1" class="reference"><a href="#cite_note-dett98-1"><span class="cite-bracket">&#91;</span>1<span class="cite-bracket">&#93;</span></a></sup><sup id="cite&#95;ref-bm01&#95;2-1" class="reference"><a href="#cite_note-bm01-2"><span class="cite-bracket">&#91;</span>2<span class="cite-bracket">&#93;</span></a></sup><sup id="cite&#95;ref-hn14&#95;3-1" class="reference"><a href="#cite_note-hn14-3"><span class="cite-bracket">&#91;</span>3<span class="cite-bracket">&#93;</span></a></sup><sup id="cite&#95;ref-5" class="reference"><a href="#cite_note-5"><span class="cite-bracket">&#91;</span>5<span class="cite-bracket">&#93;</span></a></sup><sup id="cite&#95;ref-6" class="reference"><a href="#cite_note-6"><span class="cite-bracket">&#91;</span>6<span class="cite-bracket">&#93;</span></a></sup><sup id="cite&#95;ref-7" class="reference"><a href="#cite_note-7"><span class="cite-bracket">&#91;</span>7<span class="cite-bracket">&#93;</span></a></sup> The exact solution to this problem can be formulated using <a href="/wiki/Integer_programming" title="Integer programming">integer programming</a>.<sup id="cite&#95;ref-hn14&#95;3-2" class="reference"><a href="#cite_note-hn14-3"><span class="cite-bracket">&#91;</span>3<span class="cite-bracket">&#93;</span></a></sup> Alternatively, if the number of reversed edges is very small, these edges can be found by a <a href="/wiki/Parameterized_complexity" title="Parameterized complexity">fixed-parameter-tractable algorithm</a>.<sup id="cite&#95;ref-8" class="reference"><a href="#cite_note-8"><span class="cite-bracket">&#91;</span>8<span class="cite-bracket">&#93;</span></a></sup></li>
<li>The vertices of the directed acyclic graph resulting from the first step are assigned to layers, such that each edge goes from a higher layer to a lower layer. The goals of this stage are to simultaneously produce a small number of layers, few edges that span large numbers of layers, and a balanced assignment of vertices to layers.<sup id="cite&#95;ref-dett98&#95;1-2" class="reference"><a href="#cite_note-dett98-1"><span class="cite-bracket">&#91;</span>1<span class="cite-bracket">&#93;</span></a></sup><sup id="cite&#95;ref-bm01&#95;2-2" class="reference"><a href="#cite_note-bm01-2"><span class="cite-bracket">&#91;</span>2<span class="cite-bracket">&#93;</span></a></sup><sup id="cite&#95;ref-hn14&#95;3-3" class="reference"><a href="#cite_note-hn14-3"><span class="cite-bracket">&#91;</span>3<span class="cite-bracket">&#93;</span></a></sup> For instance, by <a href="/wiki/Mirsky%27s_theorem" title="Mirsky&#39;s theorem">Mirsky's theorem</a>, assigning vertices by layers according to the length of the <a href="/wiki/Longest_path" class="mw-redirect" title="Longest path">longest path</a> starting from each vertex produces an assignment with the minimum possible number of layers.<sup id="cite&#95;ref-dett98&#95;1-3" class="reference"><a href="#cite_note-dett98-1"><span class="cite-bracket">&#91;</span>1<span class="cite-bracket">&#93;</span></a></sup><sup id="cite&#95;ref-hn14&#95;3-4" class="reference"><a href="#cite_note-hn14-3"><span class="cite-bracket">&#91;</span>3<span class="cite-bracket">&#93;</span></a></sup> The <a href="/wiki/Coffman%E2%80%93Graham_algorithm" title="Coffman–Graham algorithm">Coffman–Graham algorithm</a> may be used to find a layering with a predetermined limit on the number of vertices per layer and approximately minimizing the number of layers subject to that constraint.<sup id="cite&#95;ref-dett98&#95;1-4" class="reference"><a href="#cite_note-dett98-1"><span class="cite-bracket">&#91;</span>1<span class="cite-bracket">&#93;</span></a></sup><sup id="cite&#95;ref-bm01&#95;2-3" class="reference"><a href="#cite_note-bm01-2"><span class="cite-bracket">&#91;</span>2<span class="cite-bracket">&#93;</span></a></sup><sup id="cite&#95;ref-hn14&#95;3-5" class="reference"><a href="#cite_note-hn14-3"><span class="cite-bracket">&#91;</span>3<span class="cite-bracket">&#93;</span></a></sup> Minimizing the width of the widest layer is NP-hard but may be solved by branch-and-cut or approximated heuristically.<sup id="cite&#95;ref-hn14&#95;3-6" class="reference"><a href="#cite_note-hn14-3"><span class="cite-bracket">&#91;</span>3<span class="cite-bracket">&#93;</span></a></sup> Alternatively, the problem of minimizing the total number of layers spanned by the edges (without any limits on the number of vertices per layer) may be solved using <a href="/wiki/Linear_programming" title="Linear programming">linear programming</a>.<sup id="cite&#95;ref-gknv93&#95;9-0" class="reference"><a href="#cite_note-gknv93-9"><span class="cite-bracket">&#91;</span>9<span class="cite-bracket">&#93;</span></a></sup> <a href="/wiki/Integer_programming" title="Integer programming">Integer programming</a> procedures, although more time-consuming, may be used to combine edge length minimization with limits on the number of vertices per level.<sup id="cite&#95;ref-10" class="reference"><a href="#cite_note-10"><span class="cite-bracket">&#91;</span>10<span class="cite-bracket">&#93;</span></a></sup></li>
<li>Edges that span multiple layers are replaced by paths of dummy vertices so that, after this step, each edge in the expanded graph connects two vertices on adjacent layers of the drawing.<sup id="cite&#95;ref-dett98&#95;1-5" class="reference"><a href="#cite_note-dett98-1"><span class="cite-bracket">&#91;</span>1<span class="cite-bracket">&#93;</span></a></sup><sup id="cite&#95;ref-bm01&#95;2-4" class="reference"><a href="#cite_note-bm01-2"><span class="cite-bracket">&#91;</span>2<span class="cite-bracket">&#93;</span></a></sup></li>
<li>As an optional step, a layer of edge concentrator vertices (or confluent junctions) may be imposed between two existing vertex layers, reducing the edge density by replacing <a href="/wiki/Complete_bipartite_graph" title="Complete bipartite graph">complete bipartite</a> subgraphs by stars through these edge concentrators.<sup id="cite&#95;ref-hn14&#95;3-7" class="reference"><a href="#cite_note-hn14-3"><span class="cite-bracket">&#91;</span>3<span class="cite-bracket">&#93;</span></a></sup><sup id="cite&#95;ref-11" class="reference"><a href="#cite_note-11"><span class="cite-bracket">&#91;</span>11<span class="cite-bracket">&#93;</span></a></sup><sup id="cite&#95;ref-12" class="reference"><a href="#cite_note-12"><span class="cite-bracket">&#91;</span>12<span class="cite-bracket">&#93;</span></a></sup></li>
<li>The vertices within each layer are <a href="/wiki/Permutation" title="Permutation">permuted</a> in an attempt to reduce the number of crossings among the edges connecting it to the previous layer.<sup id="cite&#95;ref-dett98&#95;1-6" class="reference"><a href="#cite_note-dett98-1"><span class="cite-bracket">&#91;</span>1<span class="cite-bracket">&#93;</span></a></sup><sup id="cite&#95;ref-bm01&#95;2-5" class="reference"><a href="#cite_note-bm01-2"><span class="cite-bracket">&#91;</span>2<span class="cite-bracket">&#93;</span></a></sup><sup id="cite&#95;ref-hn14&#95;3-8" class="reference"><a href="#cite_note-hn14-3"><span class="cite-bracket">&#91;</span>3<span class="cite-bracket">&#93;</span></a></sup> Finding the minimum number of crossings or finding a maximum crossing-free set of edges is NP-complete, even when ordering a single layer at a time in this way,<sup id="cite&#95;ref-13" class="reference"><a href="#cite_note-13"><span class="cite-bracket">&#91;</span>13<span class="cite-bracket">&#93;</span></a></sup><sup id="cite&#95;ref-ew94&#95;14-0" class="reference"><a href="#cite_note-ew94-14"><span class="cite-bracket">&#91;</span>14<span class="cite-bracket">&#93;</span></a></sup> so again it is typical to resort to heuristics, such as placing each vertex at a position determined by finding the average or median of the positions of its neighbors on the previous level and then swapping adjacent pairs as long as that improves the number of crossings.<sup id="cite&#95;ref-dett98&#95;1-7" class="reference"><a href="#cite_note-dett98-1"><span class="cite-bracket">&#91;</span>1<span class="cite-bracket">&#93;</span></a></sup><sup id="cite&#95;ref-bm01&#95;2-6" class="reference"><a href="#cite_note-bm01-2"><span class="cite-bracket">&#91;</span>2<span class="cite-bracket">&#93;</span></a></sup><sup id="cite&#95;ref-gknv93&#95;9-1" class="reference"><a href="#cite_note-gknv93-9"><span class="cite-bracket">&#91;</span>9<span class="cite-bracket">&#93;</span></a></sup><sup id="cite&#95;ref-ew94&#95;14-1" class="reference"><a href="#cite_note-ew94-14"><span class="cite-bracket">&#91;</span>14<span class="cite-bracket">&#93;</span></a></sup><sup id="cite&#95;ref-15" class="reference"><a href="#cite_note-15"><span class="cite-bracket">&#91;</span>15<span class="cite-bracket">&#93;</span></a></sup> Alternatively, the ordering of the vertices in one layer at a time may be chosen using an algorithm that is <a href="/wiki/Parameterized_complexity" title="Parameterized complexity">fixed-parameter tractable</a> in the number of crossings between it and the previous layer.<sup id="cite&#95;ref-hn14&#95;3-9" class="reference"><a href="#cite_note-hn14-3"><span class="cite-bracket">&#91;</span>3<span class="cite-bracket">&#93;</span></a></sup><sup id="cite&#95;ref-16" class="reference"><a href="#cite_note-16"><span class="cite-bracket">&#91;</span>16<span class="cite-bracket">&#93;</span></a></sup></li>
<li>Each vertex is assigned a coordinate within its layer, consistent with the permutation calculated in the previous step.<sup id="cite&#95;ref-dett98&#95;1-8" class="reference"><a href="#cite_note-dett98-1"><span class="cite-bracket">&#91;</span>1<span class="cite-bracket">&#93;</span></a></sup><sup id="cite&#95;ref-bm01&#95;2-7" class="reference"><a href="#cite_note-bm01-2"><span class="cite-bracket">&#91;</span>2<span class="cite-bracket">&#93;</span></a></sup> Considerations in this step include placing dummy nodes on a line between their two neighbors to prevent <a href="/wiki/Bend_minimization" title="Bend minimization">unnecessary bends</a>, and placing each vertex in a centered position with respect to its neighbors.<sup id="cite&#95;ref-hn14&#95;3-10" class="reference"><a href="#cite_note-hn14-3"><span class="cite-bracket">&#91;</span>3<span class="cite-bracket">&#93;</span></a></sup> Sugiyama's original work proposed a <a href="/wiki/Quadratic_programming" title="Quadratic programming">quadratic programming</a> formulation of this step; a later method of Brandes and Köpf takes <a href="/wiki/Linear_time" class="mw-redirect" title="Linear time">linear time</a> and guarantees at most two bends per edge.<sup id="cite&#95;ref-hn14&#95;3-11" class="reference"><a href="#cite_note-hn14-3"><span class="cite-bracket">&#91;</span>3<span class="cite-bracket">&#93;</span></a></sup><sup id="cite&#95;ref-17" class="reference"><a href="#cite_note-17"><span class="cite-bracket">&#91;</span>17<span class="cite-bracket">&#93;</span></a></sup></li>
<li>The edges reversed in the first step of the algorithm are returned to their original orientations, the dummy vertices are removed from the graph and the vertices and edges are drawn. To avoid intersections between vertices and edges, edges that span multiple layers of the drawing may be drawn as polygonal chains or <a href="/wiki/Spline_curve" class="mw-redirect" title="Spline curve">spline curves</a> passing through each of the positions assigned to the dummy vertices along the edge.<sup id="cite&#95;ref-dett98&#95;1-9" class="reference"><a href="#cite_note-dett98-1"><span class="cite-bracket">&#91;</span>1<span class="cite-bracket">&#93;</span></a></sup><sup id="cite&#95;ref-bm01&#95;2-8" class="reference"><a href="#cite_note-bm01-2"><span class="cite-bracket">&#91;</span>2<span class="cite-bracket">&#93;</span></a></sup><sup id="cite&#95;ref-gknv93&#95;9-2" class="reference"><a href="#cite_note-gknv93-9"><span class="cite-bracket">&#91;</span>9<span class="cite-bracket">&#93;</span></a></sup></li></ul>
<div class="mw-heading mw-heading2"><h2 id="Implementations">Implementations</h2><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Layered_graph_drawing&amp;action=edit&amp;section=2" title="Edit section: Implementations"><span>edit</span></a><span class="mw-editsection-bracket">]</span></span></div>
<p>In its simplest form, layered graph drawing algorithms may require O(<i>mn</i>) time in graphs with <i>n</i> vertices and <i>m</i> edges, because of the large number of dummy vertices that may be created. However, for some variants of the algorithm, it is possible to simulate the effect of the dummy vertices without actually constructing them explicitly, leading to a near-<a href="/wiki/Linear_time" class="mw-redirect" title="Linear time">linear time</a> implementation.<sup id="cite&#95;ref-18" class="reference"><a href="#cite_note-18"><span class="cite-bracket">&#91;</span>18<span class="cite-bracket">&#93;</span></a></sup>
</p><p>The "dot" tool in <a href="/wiki/Graphviz" title="Graphviz">Graphviz</a> produces layered drawings.<sup id="cite&#95;ref-gknv93&#95;9-3" class="reference"><a href="#cite_note-gknv93-9"><span class="cite-bracket">&#91;</span>9<span class="cite-bracket">&#93;</span></a></sup> A layered graph drawing algorithm is also included in <a href="/wiki/Microsoft_Automatic_Graph_Layout" title="Microsoft Automatic Graph Layout">Microsoft Automatic Graph Layout</a><sup id="cite&#95;ref-19" class="reference"><a href="#cite_note-19"><span class="cite-bracket">&#91;</span>19<span class="cite-bracket">&#93;</span></a></sup> and in <a href="/wiki/Tulip_(software)" title="Tulip (software)">Tulip</a>.<sup id="cite&#95;ref-20" class="reference"><a href="#cite_note-20"><span class="cite-bracket">&#91;</span>20<span class="cite-bracket">&#93;</span></a></sup>
</p>
<div class="mw-heading mw-heading2"><h2 id="Variations">Variations</h2><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Layered_graph_drawing&amp;action=edit&amp;section=3" title="Edit section: Variations"><span>edit</span></a><span class="mw-editsection-bracket">]</span></span></div>
<p>Although typically drawn with vertices in rows and edges proceeding from top to bottom, layered graph drawing algorithms may instead be drawn with vertices in columns and edges proceeding from left to right.<sup id="cite&#95;ref-21" class="reference"><a href="#cite_note-21"><span class="cite-bracket">&#91;</span>21<span class="cite-bracket">&#93;</span></a></sup> The same algorithmic framework has also been applied to radial layouts in which the graphs are arranged in concentric circles around some starting node<sup id="cite&#95;ref-hn14&#95;3-12" class="reference"><a href="#cite_note-hn14-3"><span class="cite-bracket">&#91;</span>3<span class="cite-bracket">&#93;</span></a></sup><sup id="cite&#95;ref-22" class="reference"><a href="#cite_note-22"><span class="cite-bracket">&#91;</span>22<span class="cite-bracket">&#93;</span></a></sup> and to three-dimensional layered drawings of graphs.<sup id="cite&#95;ref-hn14&#95;3-13" class="reference"><a href="#cite_note-hn14-3"><span class="cite-bracket">&#91;</span>3<span class="cite-bracket">&#93;</span></a></sup><sup id="cite&#95;ref-23" class="reference"><a href="#cite_note-23"><span class="cite-bracket">&#91;</span>23<span class="cite-bracket">&#93;</span></a></sup>
</p><p>In layered graph drawings with many long edges, edge clutter may be reduced by grouping sets of edges into bundles and routing them together through the same set of dummy vertices.<sup id="cite&#95;ref-24" class="reference"><a href="#cite_note-24"><span class="cite-bracket">&#91;</span>24<span class="cite-bracket">&#93;</span></a></sup> Similarly, for drawings with many edges crossing between pairs of consecutive layers, the edges in maximal bipartite subgraphs may be grouped into confluent bundles.<sup id="cite&#95;ref-25" class="reference"><a href="#cite_note-25"><span class="cite-bracket">&#91;</span>25<span class="cite-bracket">&#93;</span></a></sup>
</p><p>Drawings in which the vertices are arranged in layers may be constructed by algorithms that do not follow Sugiyama's framework. For instance, it is possible to tell whether an <a href="/wiki/Undirected_graph" class="mw-redirect" title="Undirected graph">undirected graph</a> has a drawing with at most <i>k</i> crossings, using <i>h</i> layers, in an amount of time that is polynomial for any fixed choice of <i>k</i> and <i>h</i>, using the fact that the graphs that have drawings of this type have bounded <a href="/wiki/Pathwidth" title="Pathwidth">pathwidth</a>.<sup id="cite&#95;ref-26" class="reference"><a href="#cite_note-26"><span class="cite-bracket">&#91;</span>26<span class="cite-bracket">&#93;</span></a></sup>
</p><p>For layered drawings of <a href="/wiki/Concept_lattice" class="mw-redirect" title="Concept lattice">concept lattices</a>, a hybrid approach combining Sugiyama's framework with additive methods (in which each vertex represents a set and the position of the vertex is a sum of vectors representing elements in the set) may be used. In this hybrid approach, the vertex permutation and coordinate assignment phases of the algorithm are replaced by a single phase in which the horizontal position of each vertex is chosen as a sum of scalars representing the elements for that vertex.<sup id="cite&#95;ref-27" class="reference"><a href="#cite_note-27"><span class="cite-bracket">&#91;</span>27<span class="cite-bracket">&#93;</span></a></sup>
Layered graph drawing methods have also been used to provide an initial placement for <a href="/wiki/Force-based_algorithms_(graph_drawing)" class="mw-redirect" title="Force-based algorithms (graph drawing)">force-directed graph drawing algorithms</a>.<sup id="cite&#95;ref-28" class="reference"><a href="#cite_note-28"><span class="cite-bracket">&#91;</span>28<span class="cite-bracket">&#93;</span></a></sup>
</p>
